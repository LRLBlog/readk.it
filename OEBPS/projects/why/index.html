Why is readk.it needed?

Because vendor reading systems are fragmented in features and EPUB support

Means producing complex eBooks that work acceptably is difficult - often have to produce different versions for different reading systems

Vendor reading systems are finding it hard to keep up with the pace of change

EPUB3 specification is ambiguous in areas and is difficult to implement consistently

What does readk.it allow?

For publishers: More control over content presentation

Produce richer content than is possible with most vendor reading systems
Customise the reading experience for a given eBook
Customise the presentation depending on the screen size
Tap into advances in web standards and browser technology
A single digital publication that can be web-served, or distributed as an EPUB file

The ability for publishers to reach a wide audience with one EPUB file

A more consistent reading experience for a wider range of users

How does readk.it work?

Readk.it reading system is embedded in the EPUB file, and travels with it

The EPUB content is styled using Responsive Web Design techniques, allowing for suitable reading across a wide range of device sizes

The EPUB file can be read conventionally on vendor reading systems (readk.it is effectively ignored)

The EPUB file can be uncompressed for reading using a web browser

Readk.it provides the light-weight reading system to allow the EPUB content to be navigated by the user

Readk.it features can be added or removed by the publisher as desired

Readk.it can be used to present a digital reading experience online, either for a single eBook or for a library of titles
